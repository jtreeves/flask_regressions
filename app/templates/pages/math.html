Math

Here are some explanations of mathematical concepts relevant to the output from the API. Learn how to use the API to create equations, what correlation means, and which types of points reveal which types of information.

Equations

All functions can be created by using the numbers provided in the coefficients array. For example, if the linear_coefficients array is [7.64, 3.41], then that means the a value of the linear equation is 7.64 and the b value of the linear equation is 3.41.

  • Linear

    Straight line with a constant slope and a single x-intercept; best suited to situations with constant growth

    y = a\cdot{x} + b

  • Quadratic

    Parabola with either a maximum or a minimum; best suited to situations where you hope to maximize or minize a specific variable

    y = a\cdot{x^2} + b\cdot{x} + c

  • Cubic

    Curve with potentially both a maximum and a minimum; best suited to situations where you assume a variable can be maximized at one instance and minimized at another

    y = a\cdot{x^3} + b\cdot{x^2} + c\cdot{x} + d

  • Hyperbolic

    Curve that approaches an asymptote; best suited to situations where you assume the output will fall to a certain value over time

    y = a\cdot{\frac{1}{x}} + b

  • Exponential

    Curve that increases or decreases at a faster rate than any of its peers; best suited to situations where you assume the output will increase or decrease rapidly and without any limits

    y = a\cdot{b^x}

  • Logarithmic

    Curve that grows slowly; best suited to situations where you assume the output will increase at a slower-than-linear rate

    y = a\cdot{\ln{x}} + b

  • Logistic

    Curve that passes through a period of exponential growth before approaching an asymptote; best suited to situations where you assume the output will increase rapidly before levelling off

    y = \frac{a}{1 + \text{e}^{-b\cdot(x - c)}}

  • Sinusoidal

    Curve that oscillates back and forth between a maximum and minimum value; best suited to situations where you expect cyclical behavior

    y = a\cdot{\sin(b\cdot(x - c))} + d

Correlation

The correlation coefficient measures how well a model fits a data set. The closer the correlation is to 1.0, the better the fit; the closer the correlation is to 0.0, the worse the fit. There are multiple ways to calculate the correlation coefficient of a given model. This API uses the following approach:

  1. Create a list of all the observed values (i.e., the y-coordinates of all the points in the data set)

    y_i = [ y_1, y_2, \cdots, y_n ]

  2. Create a list of all the predicted values (i.e., the y-coordinates you would get for each x-coordinate if you used the created model)

    \hat{y}_i = [ \hat{y}_1, \hat{y}_2, \cdots, \hat{y}_n ]

  3. Calculate the arithmetic mean of all the observed values (i.e., the average y-coordinate from the original data set)

    \bar{y} = \frac{1}{n}\cdot{\sum\limits_{i=1}^n y_i}

  4. Create a list of all the residuals (i.e., the difference each observed value and its corresponding predicted value)

    e_i = [ y_1 - \hat{y}_1, y_2 - \hat{y}_2, \cdots, y_n - \hat{y}_n ]

  5. Create a list of all the deviations (i.e., the difference between each observed value and the mean value)

    d_i = [ y_1 - \bar{y}, y_2 - \bar{y}, \cdots, y_n - \bar{y} ]

  6. Calcuate the sum of squares of all residuals

    SS_{res} = \sum\limits_{i=1}^n e_i^2

  7. Calcuate the sum of squares of all deviations

    SS_{dev} = \sum\limits_{i=1}^n d_i^2

  8. Calcuate the correlation coefficient by taking the square root of the difference between 1 and the ratio of the sums of squares

    r = \sqrt{1 - \frac{SS_{res}}{SS_{dev}}}
