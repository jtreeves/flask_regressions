Usage

Here's a guide for how to use the API. Find out how to sign up for an account, how to create a new collection of regression models, how to get an existing collection, how to update an existing collection, how to delete an existing collection, how certain fields must be formatted, and how to use the results. Plus, see an example that walks you through all the stages of an initial POST request.

How to Sign Up

  1. Sign up with your email
  2. Store your API key somewhere safe

How to Create a New Collection

  1. Choose a source code for this collection
  2. Make a POST request
    • Include the API key that you received and the source code that you chose as parameters in the URL
    • Send the data set, the precision, the title, the indepenent variable name, and the dependent variable name as an object in the body of the request
  3. Receive an object containing the results of the regression analysis

How to Get an Existing Collection

  1. Recall your API key and what source code you previously chose for this collection
  2. Make a GET request that includes the API key and the source code as parameters in the URL
  3. Receive an object containing the results of the regression analysis

How to Update an Existing Collection

  1. Recall your API key and what source code you previously chose for this collection
  2. Make a PUT request
    • Include the API key and the source code as parameters in the URL
    • Send the data set, the precision, the title, the indepenent variable name, and the dependent variable name as an object in the body of the request (even if you only want to update one of those fields, you must send information for all of them)
  3. Receive an object containing the updated results of the regression analysis

How to Delete an Existing Collection

  1. Recall your API key and what source code you previously chose for this collection
  2. Make a DELETE request that includes the API key and the source code as parameters in the URL
  3. Receive a status of 204 to indicate that the collection was successfully deleted

How to Format Key Fields

  • URL submitted for requests {% include 'snippets/get_url.html' %}
    • API key must be the key you received when you signed up
    • Source code must be a string of alphanumeric characters less than 100-characters long
    • Source code must be unique for each of your collections (i.e., you may use 1 as a source code and another user may also use 1 as their source code, but you cannot use 1 as a source code for two separate collections)
  • Body submitted with POST and PUT requests {% include 'snippets/post_body.html' %}
    • Title must be a string (it should describe the nature of the data set)
    • Independent must be a string (it should describe the independent variable of the data set)
    • Dependent must be a string (it should describe the dependent variable of the data set)
    • Precision must be a positive integer (it determines to how many digits all answers will round)
    • Data set must be a list of lists (it is the raw data you want to analyze)
      • Outer list (i.e., the data set) must be enclosed in square brackets
      • Outer list must include at least 10 inner lists (i.e., you must provide at least 10 points)
      • Inner lists (i.e., the points) must be enclosed in square brackets
      • Inner lists must contain exactly two elements
      • Elements of inner lists (i.e., the coordinates) must be numbers
